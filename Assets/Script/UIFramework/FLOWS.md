## Flow 4: Complete Lifecycle with Caching

```
????????????????????????????????????????????????????????????????
?            COMPLETE LIFECYCLE (WITH CACHING)                  ?
????????????????????????????????????????????????????????????????

FIRST TIME:
  Show("Popup", data)
    ? Check cache: NOT FOUND
    ? Load from Addressable/Prefab
    ? Initialize(data)
    ? Add to openedViews
    ? Show (with animation)
    ? State = Visible
    
  Hide("Popup")
    ? Play hide animation
    ? State = Hidden
    ? Remove from openedViews
    ? CHECK UIConfig.enableCaching = true
    ? ADD to cachedViews["Popup"] = instance
    ? gameObject.SetActive(false)
    ? Memory: KEPT ALIVE

SECOND TIME:
  Show("Popup", data)
    ? Check cache: FOUND!
    ? Get from cachedViews
    ? Remove from cachedViews (no longer cached)
    ? Re-initialize(data) ? New data
    ? Add to openedViews
    ? Show (with animation)
    ? State = Visible
    
  Result: No loading overhead! Instant show.

MEMORY STATE:
  First Hide:  ???? Cached (inactive) ????
  Second Show: ???? Active (visible)  ????
  Second Hide: ???? Cached again      ????
```

## Flow 5: Object Pooling

```
????????????????????????????????????????????????????????????????
?                    OBJECT POOLING FLOW                        ?
????????????????????????????????????????????????????????????????

INITIALIZATION:
  UIManager.PrewarmPool("DamageNumber", 10)
    ?
    ?
  ???????????????????????????????????????
  ? Create 10 instances                  ?
  ? Set inactive                         ?
  ? Add to pool queue                    ?
  ?                                      ?
  ? Pool["DamageNumber"]:                ?
  ? [Instance1, Instance2, ... Instance10]?
  ???????????????????????????????????????

FIRST SHOW:
  Show("DamageNumber", data)
    ?
    ?
  ???????????????????????????????????????
  ? Get from pool.Dequeue()              ?
  ? Pool: [Instance2...Instance10]       ?
  ? activeInstances[Instance1] = viewId  ?
  ???????????????????????????????????????
    ?
    ?
  ???????????????????????????????????????
  ? SetParent(layer)                     ?
  ? Initialize(data)                     ?
  ? Show() ? Visible                     ?
  ???????????????????????????????????????

HIDE:
  Hide("DamageNumber")
    ?
    ?
  ???????????????????????????????????????
  ? Return to pool.Enqueue(Instance1)    ?
  ? Pool: [Instance2...Instance10, Instance1]?
  ? Remove from activeInstances          ?
  ? SetActive(false)                     ?
  ? SetParent(poolRoot)                  ?
  ???????????????????????????????????????

CONCURRENT SHOWS:
  Show("DamageNumber", data1) ? Gets Instance2
  Show("DamageNumber", data2) ? Gets Instance3
  Show("DamageNumber", data3) ? Gets Instance4
  
  All 3 visible at same time!
  Pool handles multiple instances automatically.

BENEFIT:
  - Zero Instantiate() calls after prewarm
  - Zero Destroy() calls
  - Zero GC from create/destroy
  - Predictable memory usage
```

## Key Takeaways

1. **Loading**: Async-first with race condition handling
2. **Memory**: Cache/Pool/Dispose strategies
3. **Communication**: Event-based, decoupled
4. **Data Flow**: MVVM with immutable data
5. **Injection**: Strategy pattern for animations
6. **Safety**: Weak references, cancellation tokens
7. **Performance**: Zero allocations after initialization
