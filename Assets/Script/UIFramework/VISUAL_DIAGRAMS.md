## Comparison: With vs Without Framework

### WITHOUT Framework (Traditional)

```csharp
public class ShopScreen : MonoBehaviour
{
    void OnBuyClicked()
    {
        // ? Business logic in MonoBehaviour
        if (PlayerData.Coins < item.Price)
            return;
        
        PlayerData.Coins -= item.Price;
        Inventory.AddItem(item);
        
        // ? Hard reference
        InventoryUI.Refresh();
        
        // ? Direct call
        FindObjectOfType<QuestManager>().CheckQuest();
        
        // ? Hard to test
        // ? Tightly coupled
    }
}

Problems:
- Business logic in view ?
- Hard references ?
- Not testable ?
- Tightly coupled ?
- Uses FindObjectOfType ?
```

### WITH Framework (Our System)

```csharp
// VIEW: ShopScreen.cs
public class ShopScreen : UIScreen
{
    void OnBuyClicked()
    {
        // ? Just delegate to controller
        var controller = this.controller as ShopController;
        controller?.OnBuyPressed(selectedItem);
    }
}

// CONTROLLER: ShopController.cs
public class ShopController : UIControllerBase
{
    public void OnBuyPressed(ShopItemData item)
    {
        // ? Business logic in controller
        if (!ShopService.CanBuy(item))
            return;
        
        ShopService.BuyItem(item);
        
        // ? Decouple via events
        EventBus.Instance.Publish(new ItemBoughtEvent(item));
        
        // ? Testable without Unity
        // ? Decoupled from other UIs
    }
}

Benefits:
- Clean separation ?
- Testable ?
- Decoupled ?
- SOLID compliant ?
- Production-ready ?
```

## Multiplayer Safety

```
PROBLEM: Static singleton breaks multiplayer
??????????????????????????????????????
? static UIManager instance          ? ? Shared between players!
?                                    ?
? Player 1 opens Shop                ?
? Player 2 opens Shop                ?
? ? Both access same UIManager       ?
? ? State conflicts! ?               ?
??????????????????????????????????????

SOLUTION: Instance-based managers
???????????????????????   ???????????????????????
? Player 1 Session    ?   ? Player 2 Session    ?
?                     ?   ?                     ?
? UIManager instance1 ?   ? UIManager instance2 ?
? PlayerData 1        ?   ? PlayerData 2        ?
? Shop instance 1     ?   ? Shop instance 2     ?
?                     ?   ?                     ?
? Independent! ?      ?   ? Independent! ?      ?
???????????????????????   ???????????????????????

HOW:
// Don't use singleton in multiplayer
var uiManager = new GameObject().AddComponent<UIManager>();
uiManager.Initialize(registry);
playerSession.UIManager = uiManager; // Per-player
```
