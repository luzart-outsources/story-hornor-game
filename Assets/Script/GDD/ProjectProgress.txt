💡 KEY LEARNINGS & DECISIONS
───────────────────────────────────────────────────────────────────────────────

1. Event System Architecture:
   - Chose UIFramework.EventBus over custom implementation
   - Benefit: Consistent with existing framework, zero additional dependencies
   - Trade-off: None, EventBus is robust and well-tested

2. SaveLoad Design:
   - Interface-only approach allows flexibility
   - Can implement PlayerPrefs, JSON, or binary serialization later
   - SaveData structure supports progression tracking and settings

3. Namespace Organization:
   - Luzart.Core: Shared types (enums)
   - Luzart.Game: Game-specific logic
   - Luzart.Game.Events: Centralized event definitions
   - Benefit: Clear separation, easy to navigate

4. Folder Structure:
   - Resources/ScriptableObjects: Enables runtime loading via Resources.Load
   - Prefabs/UI: Organized by UI type (Screens, Popups, Huds)
   - Benefit: Matches UIFramework patterns, scalable
